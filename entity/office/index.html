Introduction

To access the data, you can either use the manager or directly the Eloquent model.

If you wish to extend any class, remember to update the configuration. You can publish the configuration through the command php artisan vendor:publish

Let's first check the list of all attributes

Attributes

Name Fillable Required Unique Default Comment
id No No No / Indentify the entity
name Yes Yes Yes null
description Yes No No null
address_id Yes No No null
company_id Yes No No null
created_at No No No / Indicate the date when the record was created
updated_at No No No / Indicate the date when the record was last updated
deleted_at No No No / Indicate the date when the record was soft-deleted

Model

Base class Railken\Amethyst\Models\Office

You can extends the class like the following example in app/Models/Office

namespace App\Models;

use Railken\Amethyst\Models\Office as Model;

class Office extends Model {
    // ...
}

Remember to update the configuration with new class.

Manager

Base class Railken\Amethyst\Managers\OfficeManager

The manager is the main class to access and manipulate your model.

Why use the manager instead of the model? Because the manager handle all the boring stuff like validation and authorization for you. Remember that the manager return always a Result.

You can extends the class like the following example in app/Managers/OfficeManager

namespace App\Managers;

use Railken\Amethyst\Managers\OfficeManager as Manager;

class OfficeManager extends Manager {
    // ...
}

Remember to update the configuration with new class.

Schema

Base class Railken\Amethyst\Schemas\OfficeSchema

The schema is used to define the structure of the attributes. All the $attributes in the model and in the manager are initialized by the schema.

You can extends the class like the following example in app/Schemas/OfficeSchema

namespace App\Schemas;

use Railken\Amethyst\Schemas\OfficeSchema as Schema;

class OfficeSchema extends Schema {
    // ...
}

Remember to update the configuration with new class.

Repository

Base class Railken\Amethyst\Repositories\OfficeRepository

The repository is the class used to perform queries.

use Railken\Amethyst\Managers\OfficeManager;

$manager = new OfficeManager();

$repository = $manager->getRepository();

Retrieving an entity

$repository->findOneBy(['id' => 1]);
$repository->findOneById(1);

Retrieving all entities

$repository->findAll();

Performing a query using \Illuminate\DataBase\Eloquent\Builder

$repository->newQuery()->where('id', 1)->first();

You can extends the class like the following example in app/Repositories/OfficeRepository

namespace App\Repositories;

use Railken\Amethyst\Repositories\OfficeRepository as Repository;

class OfficeRepository extends Repository {
    // ...
}

Remember to update the configuration with new class.

Serializer

Base class Railken\Amethyst\Serializers\OfficeSerializer

The serializer is used to serialize an entity, you can retrieve it from the data.

use Railken\Amethyst\Managers\OfficeManager;

$manager = new OfficeManager();

$serializer = $manager->getSerializer();

And use it so serialize an entity Retrieving an entity

$entity = $repository->findOneById(1);
$serializer->serialize($entity)->toArray(); // Returns an array

You can extends the class like the following example in app/Serializers/OfficeSerializer

namespace App\Serializers;

use Railken\Amethyst\Serializers\OfficeSerializer as Serializer;

class OfficeSerializer extends Serializer {
    // ...
}

Remember to update the configuration with new class.
